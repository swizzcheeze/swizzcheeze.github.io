Fabric Patterns Documentation

A comprehensive guide to Daniel Miessler's Fabric patterns, their functionality, and usage examples.

What is Fabric?

Fabric is an AI-based tool created by Daniel Miessler that provides a library of "patterns" - specialized templates for processing different types of information using AI models. Each pattern is designed for a specific task, from summarizing content to creating visualizations or writing essays.

Fabric serves as a framework for using large language models (LLMs) more effectively by providing consistent, purpose-built templates that can be applied to various content processing tasks. Instead of crafting custom prompts for each task, users can leverage these pre-designed patterns to get consistent, high-quality outputs.

Key Features

  • 200+ Specialized Patterns: Pre-built templates for specific content processing tasks
  • Multiple AI Model Support: Works with various AI providers including OpenRouter, Gemini, and Groq
  • Consistent Output Formats: Each pattern produces structured results in specific formats
  • Command-Line Interface: Can be used from the terminal for efficient workflows
  • Extensible Design: Users can create their own patterns

How Fabric Works

Fabric operates on a simple principle:

  1. Select a Pattern: Choose from over 200 pre-designed patterns for specific tasks
  2. Provide Input: Supply the content you want to process
  3. Execute: Run the pattern through your chosen AI model
  4. Get Structured Output: Receive consistently formatted, high-quality results

This approach allows you to leverage AI for repetitive tasks while maintaining consistent quality and formatting in the outputs.

Installation and Setup

To install Fabric, follow these steps:

# Clone the repository

Writing Patterns

Patterns designed specifically for content creation and writing tasks.

write_essay

Content Creation
Essay Writing

Purpose: Writes concise, clear essays in the style of Paul Graham.

Description: This pattern generates essays focused on simplicity, clarity, and illumination of the provided topic. It mimics Paul Graham's distinctive writing style, known for clarity of thought and expression.

Example Input:

The impact of AI on knowledge work jobs

Example Output:

# The Impact of AI on Knowledge Work Jobs

Knowledge work jobs are being transformed by AI in ways we couldn't have imagined just five years ago. 

Unlike previous technological revolutions that primarily affected manual labor, AI directly impacts jobs that require thinking for a living. Accountants, lawyers, programmers, and analysts - professionals who spent years acquiring specialized knowledge - now face a tool that can replicate aspects of their expertise instantly.

What makes this shift different isn't just the speed of change, but its nature. Previous technologies automated physical tasks; AI automates thought itself...

write_micro_essay

Content Creation
Essay Writing

Purpose: Writes concise, clear, and illuminating essays on a given topic.

Description: Similar to write_essay but produces shorter, more focused pieces while maintaining the Paul Graham inspired style. Ideal for when you need impactful but brief explorations of ideas.

Example Input:

The value of time over money

Example Output:

# The Value of Time Over Money

Most people intuitively understand that time is more valuable than money, yet behave as if the opposite were true.

Unlike money, time is fundamentally non-renewable. Each person has a finite amount, and once spent, it cannot be earned back. This fundamental asymmetry should shape our decision-making more than it typically does.

The wealthiest people often optimize for time rather than additional wealth after reaching financial comfort. This isn't coincidence but recognition of a fundamental truth: the marginal utility of additional money decreases, while the value of time remains constant or increases.

write_pull-request

Technical Writing
Code Documentation

Purpose: Drafts detailed pull request descriptions for code changes.

Description: This pattern explains code changes, provides reasoning behind them, and identifies potential bugs based on git diff output. It helps maintainers understand the purpose and impact of proposed changes.

 }

Example Output:

# Make Button Component More Flexible

## Changes Made

1. Made the `onClick` prop optional by adding a `?` to the type definition
2. Added explicit type annotation to the component props destructuring
3. Added a conditional check before calling the `onClick` handler to prevent errors when not provided

## Reasoning

These changes improve the Button component's flexibility and prevent potential runtime errors:

1. Making `onClick` optional allows the Button to be used in cases where no click handler is needed (e.g., submit buttons in forms that use the form's onSubmit handler)
2. The explicit type annotation on the destructured props improves TypeScript's type checking and IDE autocompletion
3. The conditional check before calling `onClick` prevents "cannot call undefined" runtime errors if the optional prop is not provided

Pattern Categories

The 200+ patterns in Fabric can be organized into several functional categories:

Analysis Patterns

Patterns that examine, evaluate, and interpret different types of content, such as analyze_paper, analyze_prose, analyze_threat_report, and analyze_incident.

Summarization Patterns

Patterns that condense content into concise formats, including summarize, summarize_micro, create_summary, and patterns for specific content types like summarize_git_diff.

Writing Patterns

Patterns that generate various types of written content, as detailed in the section above.

Extraction Patterns

Patterns that identify and pull out specific information from content, such as extract_wisdom, extract_insights, extract_main_idea, and specialized variants.

Creation Patterns

Patterns that generate new content based on inputs, including create_visualization, create_mermaid_visualization, create_keynote, and many others.

Code-Related Patterns

Patterns specifically for working with code and development tasks, like explain_code, create_coding_project, and coding_master.

Specialized Patterns

Patterns for niche use cases like create_threat_model, find_logical_fallacies, and analyze_spiritual_text.

Using Pattern Combinations

Fabric patterns become even more powerful when used together. Consider these effective combinations:

  1. Research and Writing:
    • analyze_paper → extract_wisdom → write_essay
  2. Content Improvement:
    • extract_main_idea → improve_writing → create_summary
  3. Technical Documentation:
    • explain_code → create_mermaid_visualization → create_summary
  4. Presentation Creation:
    • extract_insights → create_keynote → improve_writing

Contributing to This Documentation

This documentation is a community effort. To contribute:

  1. Fork this repository
  2. Add or improve pattern documentation following the established format: 
    ### pattern_name

**Purpose**: Brief one-line description

**Description**: More detailed explanation

**Example Input**: Example of what you'd provide to the pattern

**Example Output**: Example of the pattern's result
  3. Submit a pull request with your changes

Documentation Standards

When documenting patterns:

  • Keep descriptions clear and concise
  • Include practical, realistic examples
  • Maintain consistent formatting
  • Verify accuracy by testing the pattern
  • Add context about when to use the pattern
